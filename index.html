01 · Modelo Conceitual

Diagrama Entidade-Relacionamento

O modelo conceitual representa o domínio de negócio de forma abstrata, identificando entidades, seus atributos e os relacionamentos entre elas.

🗃️
Entidades e Atributos
O que representar no DER
▼

O que é uma Entidade?

Uma entidade representa um objeto do mundo real sobre o qual você deseja guardar informações. No DER, é representada por um retângulo.

Exemplos: Cliente, Produto, Pedido, Funcionário, Curso.

Entidade Forte Entidade Fraca Entidade Associativa

Tipos de Atributos

🔑 Chave (PK)
Identifica unicamente cada ocorrência. Ex: CPF, ID. Representado com sublinhado no DER.
📦 Simples
Valor único e indivisível. Ex: Nome, Preço, Data.
📋 Composto
Pode ser dividido. Ex: Endereço (Rua + Número + CEP).
🔄 Multivalorado
Pode ter múltiplos valores. Ex: Telefone(s) de um cliente.
⚙️ Derivado
Calculado a partir de outros. Ex: Idade (derivada de DataNasc).

Exemplo — Entidade CLIENTE

Notação simplificada do DER
CLIENTE
id_cliente (PK) nome email cpf data_cadastro

🔗
Relacionamentos e Cardinalidades
Como conectar entidades corretamente
▼

Cardinalidades

1 : 1
Um para Um. Ex: Pessoa → Passaporte. Cada pessoa tem um passaporte.
1 : N
Um para Muitos. Ex: Cliente → Pedidos. Um cliente faz vários pedidos.
N : N
Muitos para Muitos. Ex: Aluno ↔ Disciplina. Gera uma tabela intermediária.

Exemplo de DER — E-commerce

CLIENTE ——— (1,N) ——— REALIZA ——— (1,1) ——— PEDIDO
PEDIDO ——— (1,N) ——— CONTÉM ——— (N,M) ——— PRODUTO
PRODUTO ——— (N,1) ——— PERTENCE ——— (1,N) ——— CATEGORIA
Dica para a prova: Cardinalidade N:N sempre exige uma tabela intermediária (associativa) no modelo lógico! Ex: PEDIDO_PRODUTO com quantidades e preço unitário.

02 · Modelo Lógico

Transformando DER em Tabelas

O modelo lógico traduz o modelo conceitual para estruturas de tabelas, definindo chaves primárias, chaves estrangeiras e os relacionamentos entre elas.

📐
Regras de Mapeamento
DER → Tabelas Relacionais
▼

Regras de Conversão

📦 Entidade → Tabela
Cada entidade vira uma tabela. Atributos viram colunas. Chave → PRIMARY KEY.
🔗 Rel. 1:N
A FK fica no lado N. Ex: Pedido tem id_cliente (FK).
🔁 Rel. N:N
Cria uma tabela intermediária com as FKs das duas entidades e atributos do relacionamento.
🔑 Rel. 1:1
FK fica em qualquer lado (geralmente no lado obrigatório). Pode ser UNIQUE.

Representação Textual — Modelo Lógico

NOTAÇÃO LÓGICA
CLIENTE (id_cliente, nome, email, cpf, data_cadastro)

PEDIDO (id_pedido, data_pedido, status, valor_total,
        id_cliente (FK → CLIENTE))

PRODUTO (id_produto, nome, descricao, preco, estoque,
         id_categoria (FK → CATEGORIA))

CATEGORIA (id_categoria, nome, descricao)

ITEM_PEDIDO (id_pedido (FK → PEDIDO), id_produto (FK → PRODUTO),
             quantidade, preco_unitario)
Atenção: ITEM_PEDIDO é a tabela associativa de PEDIDO × PRODUTO (N:N). A PK é composta pelas duas FKs!

03 · Modelo Físico

Scripts SQL e Documentação

O modelo físico transforma o modelo lógico em comandos SQL executáveis, com tipos de dados específicos, constraints e consistência entre todos os artefatos.

🏗️
DDL — CREATE TABLE
Comandos de definição de estrutura
▼

Tipos de Dados Comuns

Inteiros
INT, BIGINT, SMALLINT. Para IDs, quantidades, contadores.
Texto
VARCHAR(n) tamanho variável, CHAR(n) tamanho fixo, TEXT para textos longos.
Numérico
DECIMAL(p,s) ou NUMERIC para valores monetários. FLOAT para científicos.
Data/Hora
DATE (só data), TIME, DATETIME / TIMESTAMP (data + hora).
Booleano
BOOLEAN (TRUE/FALSE). Alguns SGBDs usam TINYINT(1).

Script Completo — E-commerce

SQL · DDL
-- ===== MODELO FÍSICO - E-COMMERCE =====

CREATE TABLE categoria (
  id_categoria  INT           NOT NULL AUTO_INCREMENT,
  nome          VARCHAR(80)   NOT NULL,
  descricao     TEXT,
  CONSTRAINT pk_categoria PRIMARY KEY (id_categoria)
);

CREATE TABLE cliente (
  id_cliente    INT           NOT NULL AUTO_INCREMENT,
  nome          VARCHAR(100)  NOT NULL,
  email         VARCHAR(150)  NOT NULL UNIQUE,
  cpf           CHAR(11)      NOT NULL UNIQUE,
  data_cadastro DATE          DEFAULT CURRENT_DATE,
  CONSTRAINT pk_cliente PRIMARY KEY (id_cliente)
);

CREATE TABLE produto (
  id_produto    INT              NOT NULL AUTO_INCREMENT,
  nome          VARCHAR(150)     NOT NULL,
  preco         DECIMAL(10,2)    NOT NULL,
  estoque       INT              DEFAULT 0,
  id_categoria  INT              NOT NULL,
  CONSTRAINT pk_produto    PRIMARY KEY (id_produto),
  CONSTRAINT fk_prod_cat   FOREIGN KEY (id_categoria)
                            REFERENCES categoria(id_categoria)
);

CREATE TABLE pedido (
  id_pedido     INT           NOT NULL AUTO_INCREMENT,
  data_pedido   DATE          NOT NULL,
  status        VARCHAR(20)   DEFAULT 'PENDENTE',
  valor_total   DECIMAL(12,2),
  id_cliente    INT           NOT NULL,
  CONSTRAINT pk_pedido      PRIMARY KEY (id_pedido),
  CONSTRAINT fk_ped_cli     FOREIGN KEY (id_cliente)
                             REFERENCES cliente(id_cliente)
);

CREATE TABLE item_pedido (
  id_pedido      INT            NOT NULL,
  id_produto     INT            NOT NULL,
  quantidade     INT            NOT NULL,
  preco_unitario DECIMAL(10,2)  NOT NULL,
  CONSTRAINT pk_item          PRIMARY KEY (id_pedido, id_produto),
  CONSTRAINT fk_item_ped      FOREIGN KEY (id_pedido)
                               REFERENCES pedido(id_pedido),
  CONSTRAINT fk_item_prod     FOREIGN KEY (id_produto)
                               REFERENCES produto(id_produto)
);

04 · Consultas SQL

SELECT, JOIN e Condições

As consultas DML são o coração do uso prático de bancos de dados. Domine SELECT, filtros, agrupamentos e JOINs.

⚡
SELECT — Estrutura Completa
Ordem de execução e cláusulas
▼

Ordem de Escrita vs Ordem de Execução

1. FROM
Define a(s) tabela(s) de origem. Executado primeiro.
2. JOIN
Combina tabelas relacionadas.
3. WHERE
Filtra linhas antes do agrupamento.
4. GROUP BY
Agrupa linhas com valores iguais.
5. HAVING
Filtra grupos após o GROUP BY.
6. SELECT
Define quais colunas/expressões retornar.
7. ORDER BY
Ordena o resultado final. ASC ou DESC.
8. LIMIT
Limita a quantidade de linhas retornadas.

Exemplos de Consultas

SELECT BÁSICO
-- Todos os clientes cujo nome começa com 'A'
SELECT id_cliente, nome, email
FROM   cliente
WHERE  nome LIKE 'A%'
ORDER BY nome ASC;

-- Produtos com preço entre 50 e 200 reais
SELECT nome, preco
FROM   produto
WHERE  preco BETWEEN 50 AND 200
ORDER BY preco DESC;

-- Pedidos com status PENDENTE de um cliente específico
SELECT id_pedido, data_pedido, valor_total
FROM   pedido
WHERE  id_cliente = 5
  AND  status = 'PENDENTE';
GROUP BY + HAVING + FUNÇÕES DE AGREGAÇÃO
-- Total gasto por cliente (apenas quem gastou mais de R$500)
SELECT
  c.nome,
  COUNT(p.id_pedido)        AS qtd_pedidos,
  SUM(p.valor_total)        AS total_gasto,
  AVG(p.valor_total)        AS ticket_medio
FROM   cliente c
JOIN   pedido  p ON p.id_cliente = c.id_cliente
GROUP BY c.id_cliente, c.nome
HAVING SUM(p.valor_total) > 500
ORDER BY total_gasto DESC;
🔀
Tipos de JOIN
INNER, LEFT, RIGHT, FULL
▼

Comparativo de JOINs

INNER JOIN
Retorna só os registros que têm correspondência em ambas as tabelas. O mais comum.
LEFT JOIN
Retorna todos da tabela esquerda + os correspondentes da direita. Sem correspondência → NULL.
RIGHT JOIN
Retorna todos da tabela direita + os correspondentes da esquerda.
FULL JOIN
Retorna todos de ambas as tabelas, com NULL onde não há correspondência.

Exemplos de JOIN

JOINS NA PRÁTICA
-- INNER JOIN: pedidos com dados do cliente
SELECT p.id_pedido, c.nome, p.data_pedido, p.valor_total
FROM   pedido  p
INNER JOIN cliente c ON c.id_cliente = p.id_cliente;

-- LEFT JOIN: todos os clientes, mesmo sem pedido
SELECT c.nome, p.id_pedido, p.data_pedido
FROM   cliente c
LEFT JOIN pedido p ON p.id_cliente = c.id_cliente;

-- JOIN com 3 tabelas: itens com produto e pedido
SELECT
  c.nome         AS cliente,
  pr.nome        AS produto,
  i.quantidade,
  i.preco_unitario
FROM   item_pedido i
JOIN   pedido      p  ON p.id_pedido  = i.id_pedido
JOIN   cliente     c  ON c.id_cliente = p.id_cliente
JOIN   produto     pr ON pr.id_produto = i.id_produto
WHERE  p.data_pedido >= '2024-01-01';

Subqueries

SUBCONSULTAS
-- Produtos mais caros que a média geral
SELECT nome, preco
FROM   produto
WHERE  preco > (SELECT AVG(preco) FROM produto);

-- Clientes que NÃO fizeram pedido (NOT IN)
SELECT nome, email
FROM   cliente
WHERE  id_cliente NOT IN (
  SELECT DISTINCT id_cliente FROM pedido
);

FROM     tabela

INNER JOIN  → interseção (A ∩ B)
LEFT  JOIN  → tudo de A + B quando tem
RIGHT JOIN  → tudo de B + A quando tem
